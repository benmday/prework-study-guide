HTML

  • Elements are the different parts of code that serve unique funtions, enclosed in tags "< >"
  • head elements contain backend code, or "metadata," that fromtend users typically do not see
  • body elements contain frontend code that the user sees
  • Attributes define more info about the elements, and MUST be defined at the beginning of an element, never the end

CSS

  • CSS - Cascading Style Sheets
  • Inline CSS - styles an element directly in the HTML file; useful for quick and permanent change, but is less flexible
  • Internal Style Sheet - contains CSS rules in the < head > section of the HTML file; applies to all tagged elements on that page
  • External Style Sheet - a separate file contains all CSS rules; makes design changes easier; link the file using < link > element in the HTML < head > element
  • Each HTML element is a box, referred to as the CSS Box Model with margin, border, padding, and content
  • Margin indicates how much space is around the outside of an element
  • Padding indicates how much space is around the content inside and element
  • DRY Code - Do not Repeat Yourself Code; applies the same rules to multiple elements to decrease size and complexity of code

Git

  • "Repo" or Repository: project folder that contains all files and revision history
  • Branch: workspace to make project changes before going live; "testing ground"
  • Markdown (.md): file format in Github; plain-text format that translates easily to HTML
  • Commit: saving a new version of the file you're working on
  • Pull Request (PR): merging changes from one branch to another; in the request you can review your work together with your team
  • Merge: PR is approved and changes are merged to the main branch
  • Issue: report new tasks/fixes; progress is tracked on the project board; "to-do" list
  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switched to it
  • git switch: switched the branch you are working on

JavaScript

  • JavaScript (JS) is a high-level, dynamic programming language that allows users to interact with websites
  • It programs effects, interactivity, data communication, etc.
  • A Variable (var followed by unique variable name) is a container that allows us to store data in our code
  • Control Flow is the order in which the code is read; JS reads code top to bottom
  • Conditional statements and loops are used to control the order code is executed in
  • An Array is a single variable used to hold a group of data inside
  • Index is the location of data inside an array; starts with value 0
  • A For loop uses indexes to perform a task on all items in an array
  • 1. Starting point of the loop (var x=0;)
  • 2. Condition (x < array-name.length;)
  • 3. Allows array to iterate over each index (x++ is used for x+1 - each time code loops, index increased by 1)
  • A Function is a set of instructions for how to perform a certain task
  • -Functions do not automatically execute, they must be called by name
  • -Functions can be called multiple times in the same code
  • -First you define the function, then you call the function
